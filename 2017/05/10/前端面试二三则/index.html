前端面试“二三事”

一、面试前先了解的产品

　　一场应聘从你接到面试邀请就已经开始了。第一件事情就要了解一下应聘公司的产品。注册帐号使用一下对方公司的产品，打开控制台看一下的技术选型、代码水平，对于有经验的开发者，看代码是最直接的沟通方式了。可以判断一下以后的工作难度和方向，在面试中也可以成为自己主动出击的方向：找几个GUB，提几个有建议性的改进意见，都可以为自己加分不少。

二、一定要带上自己的项目作品

　　github 上的开源项目或者公司项目链接，哪怕几张产品截图也是要有的。千万不要在你的项目经验里填写“仿淘宝页面”或者“仿饿了么APP”这种非原创项目了，我的意思是不要抄袭那些公开的和自身经历无关的项目。对于那些没有工作经验的新人来说，最好能准备一个有点自己想法又想表现自己技术水平的项目，比如个人博客或者网站，这样至少能表现出一些诚意。

三、理解面试官的意图

　　一般技术面试提问大约可以分为两个方向，一类是喜欢问各种技术基本问题，比如用过几个框架、面向对象的几种写法、手机端页面优化方法、css兼容性问题等等，这种网上几乎都能找到大把现成的的答案，对于喜欢刷面试题的同学都没有太大的难度。另一类面试官比如像我这样的，喜欢抓着你的工作项目问到底，从设计到交付流程，从项目目录到打包测试，从底层架构到后期重构，一路上的各种坑是怎么解决的？这就更看注你的个人经验和现场发挥了。无论是怎样的面试问题吧，答题都离不开技术和策略两方面，技术问题就是你从网上能知道的那些，面向对象有N种写法，但这很可能并不是面试官想要的。所谓“策略”就是你自己在实战中的权衡和取舍，简单回答完技术问题之后，一定要回到自己的项目中总结一下——为什么在组件开发中使用“组合继承”比“类继承”更合适呢？这才是面试官真正想听到的答案，而不是那些照本宣科的答案。

四、学会面试你的面试官

　　应聘是双方的，我们在应聘的时候同时也是在考查应聘公司的水准。面试官的提问最能够代表对方公司的水准。好的问题既需要有一定的衡量标准，又要能给应聘者一点发散的空间。比如默写ajax考查api 或者提问前端发展方向这类又大又空的题就是很失水准的问题了。在面试之后也要总结一下对方的水平是不是合适自己的发展。另外，有时面试官会给出面试者一些提问的机会，就我而言，一般会比较喜欢应聘者反问我们当前的项目的面临的技术难关，善于提问也可以挖到不少有用的信息。

另外几个要注意的问题：

　　1.不要过分强调自己勤备好学，自学能力很强，也不要表达特别期待有大牛在工作中带带自己， 这都是一种”学生思维“；
　　2.切忌花式炫耀自己的技术能力，尤其是一些技术社区里刚听来的新概念新名词，会显得特别浮躁；
　　3.不要在工作中带入自己技术倾向，比如”react大法好“，”老夫就用jQuery“，这种技术狂热很容易让面试官警惕今后的团队合作风险；
　　4.不要在简历中堆砌各种技术框架，“精通*”等十几种框架和工具，工作经历越丰富的人，越明白自己的局限和不足；
　　5.多说说自己在工作中解决过什么样的问题，而不是吹嘘又使用了什么流行的新工具，比如说解决过“按需加载”的问题，要比说自己用过webpack/gulp更实际。

　　简单总结下能想到的几点，都是我自己在面试过程中碰到过的现象和问题。希望对你有用，帮助你找到心怡的工作。
